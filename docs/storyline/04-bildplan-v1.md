# Bildplan V1

Dieser Ordner soll spaeter nicht nur Texte enthalten, sondern auch Bilder,
Screenshots und Material fuer die Slides.

## Grundsatz

Fuer diese Praesentation sind persoenliche, dokumentarische Bilder staerker als
glatte Stock-Bilder.

Ideal sind:

- eigene Fotos
- Screenshots echter Projekte
- einfache Dokumente / Skizzen / Karten
- Material mit Geschichte, nicht nur mit Deko-Wert

## Ablageort

Bitte alle neuen Bilder in:

`docs/storyline/assets/`

## Empfohlene Dateibenennung

Bitte spaeter moeglichst so benennen:

- `slide-04-altstaetten.jpg`
- `slide-08-fhgr-chur-collage.jpg`
- `slide-13-ta-interaktiv-screenshot.png`
- `slide-16-remote-konzept-2016.jpg`
- `slide-17-kapstadt-muizenberg.jpg`
- `slide-21-vanlife-oder-haus.jpg`
- `slide-23-tnh-projekt-01.png`

So koennen wir Bilder spaeter einfacher ins Deck mappen.

## Bildwuensche pro Kapitel

### Herkunft / Altstaetten / Lehre

Moegliche Assets:

- altes Familienfoto oder Jugendbild
- Altstaetten / Rheintal / Karte
- etwas, das Lehre / fruehe Berufswelt visualisiert

### FHGR / Chur / erste Jobs

Moegliche Assets:

- FHGR
- Chur
- fruehe Agentur- oder Praktikumsarbeiten
- Filmset / Kamera / Schnittplatz
- altes Webprojekt oder Code-Screenshot

### Tages-Anzeiger / Interaktiv

Moegliche Assets:

- Screenshot von `interaktiv.tagesanzeiger.ch`
- 1-2 konkrete Projekte aus dem Team
- eventuell ein Bild vom Newsroom oder aus der Zeit in Zuerich

### Remote-Konzept / Kapstadt

Moegliche Assets:

- das 2-Seiten-Konzept an den Chef
- Weltkarte / Zeitzonen-Skizze
- Muizenberg / Strand / Berge
- fruehes Kapstadt-Foto
- optional ein persoenliches Bild mit Zoe, falls dir das fuer diese Buehne stimmt

### Freelancing / Reisen / Van / Haus

Moegliche Assets:

- Vanlife-Foto
- Reisefoto, das nicht beliebig wirkt
- Haus in Suedafrika, falls du das zeigen willst
- Arbeitssituation unterwegs

### The New Humanitarian / Non-Profit-Nische

Moegliche Assets:

- TNH-Screenshot
- Drupal / Storytelling-Projekt
- `duby.io/portfolio`-Beispiele
- Visual, das Accessibility oder globales Arbeiten greifbar macht

## Beste Bildtypen fuer die bestehenden Komponenten

### `title`

- starkes persoenliches Bild
- eher ruhig, nicht zu busy

### `twoColumn`

- Screenshots von Projekten
- Karten
- Dokumente
- Vorher/Nachher

### `media`

- heroische Einzelbilder
- Landschaft / Stadt / Reise
- ein starkes Projektbild

### `quote`

- meist kein Bild noetig
- wirkt stark mit viel Weissraum

## Material, das besonders spannend waere

Wenn vorhanden, waeren diese Assets gold wert:

- Screenshot oder Foto vom alten Remote-Konzept
- fruehe Webarbeit aus Teenagerzeit oder Studium
- Interaktiv-Projekt, auf das du wirklich stolz bist
- erstes Bild aus Muizenberg / Kapstadt
- ein TNH-Projekt, das Storytelling und Sinn stark zeigt

## Was wir spaeter damit machen koennen

Sobald Bilder in diesem Ordner liegen, kann ich:

- sie per `npm run optimize:storyline-images` in `docs/storyline/assets/` als `.webp` konvertieren
- passende Dateien fuer einzelne Slides auswaehlen
- sie ins Codebase uebernehmen
- Alt-Texte formulieren
- Slide-Layout an Bildformat und Gewicht anpassen
- Screenshots als `media` oder `twoColumn` sauber ins Deck einbauen
